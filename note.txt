Build a Chrome Extension (Manifest V3) that reads NEW incoming WhatsApp Web messages out loud automatically.

Goal / Behavior:
- When I am on https://web.whatsapp.com/, the extension should detect each new incoming message and speak it using the browser SpeechSynthesis API.
- Speak only INCOMING messages (messages from other people), NOT my own outgoing messages.
- Speak only the latest new message once (avoid repeating when scrolling, switching chats, or DOM re-renders).
- It should work even if WhatsApp Web updates its DOM often; use MutationObserver + smart filtering + a dedup cache.
- Support group chats: speak like: "[GroupName] [SenderName]: message".
- Support direct chats: speak like: "[ContactName]: message".
- Ignore system messages like "Messages are end-to-end encrypted", "You deleted this message", "Missed voice call", timestamps, etc (filter heuristics).
- Add a simple extension popup UI:
  - Toggle ON/OFF
  - Voice dropdown (list available voices)
  - Rate slider (0.5–2.0)
  - Pitch slider (0–2.0)
  - Volume slider (0–1.0)
  - Checkbox: Speak group name
  - Checkbox: Speak sender name
- Persist settings using chrome.storage.sync.

Technical requirements:
- Use Manifest V3 with:
  - content_script running on web.whatsapp.com
  - background service worker if needed
  - permissions: storage, activeTab, scripting (only if necessary)
  - host_permissions: https://web.whatsapp.com/*
- Content script:
  - Use MutationObserver on the message container.
  - Find the newest message bubble reliably.
  - Detect if it is incoming vs outgoing using class/aria labels or structure; implement fallback heuristics.
  - Create a unique message id (chat id + sender + text + timestamp if available) and store in an in-memory Set with max size (e.g., 500) to prevent duplicates.
  - When a new message is detected and extension is enabled, call window.speechSynthesis.speak(...) with current settings.
- Provide all files:
  - manifest.json
  - content.js
  - popup.html
  - popup.js
  - popup.css (optional)
  - background.js (only if required)

Deliverables:
1) Full code for each file.
2) Clear install instructions: load unpacked, pin extension, open WhatsApp Web.
3) Notes on how to update selectors if WhatsApp changes DOM.
